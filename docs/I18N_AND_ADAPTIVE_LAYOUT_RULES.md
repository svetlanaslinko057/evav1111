# i18n Translation & Adaptive Layout Rules — CANONICAL CONTRACT

**Це обов'язковий документ для будь-якого агента або розробника, який торкається перекладів або UI-розкладок у цьому проєкті.**

Будь-який PR, що порушує ці правила, **повинен бути відхилений** при code review.

---

## ⚠️ КОНТЕКСТ — чому ці правила існують

У попередніх ітераціях агенти перекладали `Billing` → `Білінг` (транслітерація),
що **неправильно**: в українській мові поняття «білінг» не існує як терміна.
Правильно — `Платежі`, `Оплата` або `Рахунки` залежно від контексту.

Аналогічно, агенти не враховували, що українські слова **довші за англійські
на 30–50%**, через що верстка ламалась при перемиканні мови (заголовок
обрізався, кнопки заходили за межі екрана).

Ці правила усувають обидві проблеми **системно**.

## 📐 ПРАВИЛО 1 — Семантичний переклад, не транслітерація

### ❌ ЗАБОРОНЕНО

Транслітерувати англійські технічні терміни кирилицею, коли в українській
мові існує природне слово:

| EN | ❌ Транслітерація | ✅ Правильно |
|---|---|---|
| Billing | Білінг | **Платежі** / Оплата / Рахунки |
| Dashboard | Дашборд | **Панель** / Огляд / Головна |
| Login | Логін | **Вхід** |
| Submit | Сабміт | **Надіслати** / Подати |
| Cancel | Кенсел | **Скасувати** |
| Settings | Сетінгс | **Налаштування** |
| Profile | Профайл | **Профіль** (тут транслітерація прижилась — ОК) |
| Onboarding | Онбординг | **Початок роботи** / Вступ |
| Workflow | Воркфлоу | **Робочий процес** |
| Feedback | Фідбек | **Відгук** (короткий) / Зворотний зв'язок (довгий) |
| Deadline | Дедлайн | **Термін** / Кінцевий строк |
| Pipeline | Пайплайн | **Конвеєр** / Потік / Воронка |
| Checkpoint | Чекпоінт | **Контрольна точка** |
| Workspace | Воркспейс | **Робочий простір** |

### ✅ ДОЗВОЛЕНО зберігати англійською (бренди + усталені IT-терміни)

- **Брендові імена**: EVA-X, Stripe, PayPal, GitHub, Telegram, Cloudinary, Resend
- **Усталені кириличні** (вже стали частиною мови): Email, IP, URL, SaaS, API, QR-код
- **Технічні короткі ідентифікатори в UI**: SEQ-01, UTC+3, v1.0
- **Імена власні**: Acme Inc., Jane Doe

### 🧭 Як вирішити: транслітерувати чи перекладати?

1. **Чи існує українське слово, яке носій мови використовує в розмові?**
   - Так → використати його (Billing → Платежі)
   - Ні → залишити англійською (API → API)
2. **Чи це бренд/назва продукту?** → залишити англійською
3. **Чи це довгий технічний термін, де є зрозумілий український аналог?** → переклад
4. **Сумніваєшся?** → подивитись на офіційні переклади Apple/Google/Microsoft українською

## 📐 ПРАВИЛО 2 — Адаптивна верстка під різну довжину слів

Українські слова в середньому **на 30–50% довші за англійські**:

| EN | UK | Δ символів |
|---|---|---:|
| Home | Головна | +75% |
| Activity | Активність | +25% |
| Profile | Профіль | 0% |
| Billing | Платежі | +14% |
| Save | Зберегти | +100% |
| Cancel | Скасувати | +56% |
| Settings | Налаштування | +50% |

### Обов'язкові правила для всіх Text-елементів, що можуть переключатися мовою

#### 2.1. Заголовки сторінок (h1, page titles)

```tsx
// ✅ ПРАВИЛЬНО
<Text
  style={styles.h1}
  numberOfLines={1}
  adjustsFontSizeToFit
  minimumFontScale={0.7}
>
  Activity
</Text>

// styles:
h1: {
  fontSize: 32,
  fontWeight: '800',
  flexShrink: 1,    // ← КРИТИЧНО для flex-row контейнерів
  minWidth: 0,      // ← КРИТИЧНО, інакше flexShrink не працює
}
```

#### 2.2. Кнопки (button labels)

```tsx
// ✅ Великі кнопки (full-width) — auto-shrink текст
<TouchableOpacity style={styles.btn}>
  <Text
    style={styles.btnLabel}
    numberOfLines={1}
    adjustsFontSizeToFit
    minimumFontScale={0.85}
  >
    Save changes
  </Text>
</TouchableOpacity>

// ✅ Маленькі кнопки (фіксованої ширини) — використовуй коротший переклад
// + nowrap через numberOfLines={1}
```

#### 2.3. Боттом-таби (5 пунктів)

```tsx
tabBarLabelStyle: {
  fontSize: 11,           // ← маленький розмір дозволяє довгим словам вмиститись
  // НЕ ставити fontWeight: '800' (товсте — займає більше місця)
}
// AND choose SHORT Ukrainian translations for tabs (1–2 short words max)
// e.g. "Home" → "Головна" (7 chars OK), не "Головна сторінка"
```

#### 2.4. Заголовки з pill/badge поряд (header + project pill)

```tsx
// ✅ ПРАВИЛЬНО — heading shrinks, pill stays bounded
<View style={styles.headerRow}>
  <Text style={styles.h1} numberOfLines={1} adjustsFontSizeToFit minimumFontScale={0.7}>
    Activity
  </Text>
  <View style={styles.pill}>
    <Text numberOfLines={1}>Acme Analytics Platform</Text>
  </View>
</View>

// styles:
headerRow: { flexDirection: 'row', gap: 12, alignItems: 'flex-end' },
h1:   { ..., flexShrink: 1, minWidth: 0 },
pill: { ..., flexShrink: 0, maxWidth: 180 },  // pill каже свій максимум
```

#### 2.5. Списки з лейблом + значенням (key-value rows)

```tsx
// ✅ Якщо label довгий — дозволити йому wrap, або скоротити переклад
<View style={styles.row}>
  <Text style={[styles.label, { flexShrink: 1 }]} numberOfLines={2}>
    {tByEn('Estimated total')}
  </Text>
  <Text style={[styles.value, { flexShrink: 0 }]}>$5,200</Text>
</View>
```

### Загальні принципи

1. **Будь-який Text у flex-row контейнері** → `flexShrink: 1` + `minWidth: 0` + `numberOfLines={N}`
2. **Великі заголовки** (h1, h2) з фіксованим шрифтом → `adjustsFontSizeToFit minimumFontScale={0.7}`
3. **Pill / badge / chip** в одному ряду з заголовком → `flexShrink: 0` + `maxWidth: N`
4. **Уникати фіксованих `width`** для текстових контейнерів — використовуй `maxWidth`
5. **Тестувати верстку обома мовами** (EN + UK) перед merge — це обов'язково

## 📐 ПРАВИЛО 3 — Перевірка перед commit

Перед будь-яким PR, що змінює переклад або UI з текстом:

```bash
# 1. Перевірка parity dictionaries
python3 /app/scripts_repo/cabinet-i18n-coverage.py
# Має бути: hardcoded≈0  en==uk

# 2. Візуальна перевірка UK на основних екранах
# - Welcome / landing
# - Auth / sign-in
# - Home / dashboard
# - Activity (заголовок + pill з довгою назвою проєкту)
# - Settings
# - Bottom tabs (всі 5)

# 3. Перевірка на транслітерацію
# Якщо в diff є слова типу: Білінг, Дашборд, Сабміт, Логін —
# це червоний прапор. Знайти український аналог.
```

## 📐 ПРАВИЛО 4 — Контекстуальні переклади

Одне англійське слово може мати **різні переклади** залежно від контексту:

| EN | Контекст | UK |
|---|---|---|
| "Approve" | модерація задачі | **Схвалити** |
| "Approve" | згода на витрату | **Підтвердити** |
| "Submit" | надсилання форми | **Надіслати** |
| "Submit" | подання на QA | **Подати** |
| "Active" | користувач онлайн | **Онлайн** / Активний |
| "Active" | стан модуля | **В роботі** |
| "Done" | модуль завершено | **Завершено** |
| "Done" | ОК на кнопці | **Готово** |

→ Завжди читай контекст, не перекладай ізольовано.

## 📁 Де живуть словники

- **Web cabinet**: `/app/web/src/i18n/dictionary.js` — `DICTIONARY.en` + `DICTIONARY.uk`
- **Mobile Expo**: `/app/frontend/src/i18n.tsx` — `EN: Dict` + `UK: Dict`
- **Backend system copy**: `/app/backend/i18n_backend.py` — `_DICT['en']` + `_DICT['uk']`

Усі троє підтримують `parity` — кількість EN-ключів = UK-ключів. Скрипт
`/app/scripts_repo/cabinet-i18n-coverage.py` верифікує це.

## 🚦 Чеклист для агентів

Перед додаванням нового UK перекладу:

- [ ] Я перевірив, що це не транслітерація (Білінг, Дашборд, Сабміт ❌)
- [ ] Я використав слово, яке вживає носій мови в розмові
- [ ] Я зважив контекст (модерація vs згода vs стан)
- [ ] Я переконався, що UK переклад не довший за EN на >50% (інакше — або скоротити, або додати `adjustsFontSizeToFit`)
- [ ] Я перевірив верстку: heading + pill + bottom tabs не ламаються

Перед додаванням нового Text-елементу в JSX:

- [ ] Він обгорнутий у `<Text>` з `@/src/i18n-text` (для mobile) або `tByEn(...)` (для web)
- [ ] Якщо він у flex-row — є `flexShrink: 1` + `minWidth: 0` + `numberOfLines={N}`
- [ ] Якщо це заголовок з можливим переповненням — є `adjustsFontSizeToFit`
- [ ] Я перевірив візуально на EN + UK перед commit
